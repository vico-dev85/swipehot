   - Query xcam_knowledge_base for relevant entries

## Internal Linking System

### Technical Approach: Hybrid Static + Dynamic

**Static Links (Generated at creation time):**

**Dynamic Links (Queried at page load):**

### Related Models Query
```sql
-- Find similar models
SELECT model_name, display_name, wordpress_post_id
FROM xcam_models
WHERE status = 'active'
  AND categories && ARRAY['latina', 'interactive'] -- overlap with MiaRose's categories
  AND model_name != 'miarose' -- exclude self
ORDER BY num_followers DESC
LIMIT 8;
```

```php
// In model-page template

$model_name = get_post_meta(get_the_ID(), 'model_name', true);
$categories = get_post_meta(get_the_ID(), 'categories', true);

### **8. SEO Risk Mitigation** ⭐ IMPORTANT
**Currently:** Not mentioned
**Need:** Avoid Google penalties

```markdown
## SEO Risk Management

### Thin Content Prevention
**Risk:** Google penalizes pages with little unique value

**Mitigation:**
1. Minimum 300 words per model page
2. Each page has unique angle/focus
3. Variation in templates (not identical structure)
4. Add unique sections: "Viewer Reviews", "Show Highlights", etc.

### Duplicate Content Prevention
**Risk:** Similar model pages look duplicate

**Mitigation:**
1. Check similarity score before publishing (< 70% match)
2. Vary writing style per category
3. Use different sentence structures
4. Include unique facts per model (streaming schedule, specialties)

### Over-Optimization Prevention
**Risk:** Keyword stuffing, unnatural linking

**Mitigation:**
1. Model name max 5 times per 400 words
2. Internal links feel natural, not forced
3. Varied anchor text (not repetitive)
4. Link density cap: 1 per 100 words

### Indexing Strategy
**Don't:** Generate 1,000 pages and submit all at once

**Do:**
1. Week 1: 20 pages
2. Week 2: 50 pages
3. Week 3: 100 pages
4. Monitor: Watch for indexing issues
5. Scale: Gradually increase if no problems

### Quality Signals
- Fast page load (< 2 seconds)
- Mobile-optimized
- Valid HTML
- Proper schema markup
- Real user engagement (time on page)
```
